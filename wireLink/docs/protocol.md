# 协议解析说明

## Modbus RTU

串口固定 8N1、无流控。支持 03H 和 06H；CRC16 低字节在前。03H 单次寄存器数量限制 1～125。每帧校验从机地址、功能码、字节数、总长度和 CRC。超时或 CRC 错误重试一次；02/03/04 等设备异常直接返回。

默认请求超时 1000 ms。RTU 请求间应用至少 3.5 字符静默期。设备地址暂允许 1～255；100～255 显示范围未确认提示，地址 0 不读。

## 地址行规则

- 下一行只有地址：该地址是上一逻辑字段的附加寄存器，例如 336/337 只产生“高精度电流 Ia”一个显示项。
- 下一行地址为空：仍是同一寄存器内的字节或位段含义，不发起额外读取。
- 地址数组保持协议顺序，不能排序；当前报警是 `[514,513]`。

## 数值规则

- 单寄存器内部高字节在前。
- uint32 默认 `HighWordFirst`，可切换 `LowWordFirst`；在实机确认前显示黄色字序警告并保留两个原始寄存器到 Debug 日志。
- `×N` 为原值乘 N；百分数字段为原值除以 100，显示两位小数。
- `×电流变比` 使用隐藏寄存器 788；788 不显示、不导出。
- `×1 或 ×2 注1` 暂只显示十六进制原值和倍率待确认警告。
- 非法 BCD、未知枚举和保留值只影响当前字段，不使整批失败。

## 读取区间

设备：256～258、268～270、336～343、352～357、432～433、512～523，加隐藏 788。连接测试读取 256 一个寄存器。

故障：06H 写 785，值为 `(序号 << 8) | 类型`，校验回显，等待高级设置的 0～2000 ms（默认 100 ms），再读 768～788。只展示/导出 768～787。

变位记录仅解析时间与通用字段；事件数据保持原值并警告。根目录 README 的“待补充协议规则”是剩余问题清单。
