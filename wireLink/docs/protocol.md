# 协议解析说明

## Modbus RTU

串口固定 8N1、无流控。支持 03H 和 06H；CRC16 低字节在前。03H 单次寄存器数量限制 1～125。每帧校验从机地址、功能码、字节数、总长度和 CRC。超时或 CRC 错误重试一次；02/03/04 等设备异常直接返回。

默认请求超时 1000 ms。RTU 请求间应用至少 3.5 字符静默期。设备地址由用户在 1～255 内输入；地址不正确或没有对应设备时，连接测试按通信失败处理。地址 0 不用于读操作。

## 地址行规则

- 下一行只有地址：该地址是上一逻辑字段的附加寄存器，例如 336/337 只产生“高精度电流 Ia”一个显示项。
- 下一行地址为空：仍是同一寄存器内的字节或位段含义，不发起额外读取。
- 地址数组保持协议顺序，不能排序；当前报警是 `[514,513]`。

## 数值规则

- 单寄存器内部高字节在前。
- uint32 的寄存器顺序已确认：定义中排在前面的寄存器是高 16 位，后面的是低 16 位。例如 336 高、337 低；当前报警按协议表顺序为 514 高、513 低。默认使用 `HighWordFirst`；高级设置保留 `LowWordFirst` 仅用于诊断，Debug 日志始终记录两个原始寄存器。
- `×N` 为原值乘 N；百分数字段直接使用原始值，显示为紧邻数值的 `%`，例如原值 35 显示为 `35%`。
- 电流变比只允许 1 或 2。BW1 的额定电流序值 ≤10 时为 1，否则为 2；BW3 的序值 ≤11 时为 1，否则为 2。
- 当前暂按寄存器 787 返回“额定电流序值”实现。用户在高级通信设置中选择 `BW1 的控制器` 或 `BW3 的控制器`，软件用相应的额定电流表把序值映射为安培值，再按上述阈值计算电流变比。
- `CurrentRatioRule` 固化 BW1/BW3 的序值—额定电流表和变比边界。序值超出所选控制器的表格范围时，该字段显示原值和解析警告。
- 5.5 中“×1 或 ×2 注1”的电流事件也使用“控制器系列 + 787 序值”计算；寄存器 788 不参与计算。漏电仍为 `×0.01`，频率为 `×0.01`，百分比使用原值。
- 这是实机连接前的暂定解释；如果设备实际在 787 返回额定电流安培值，应根据实机结果调整。
- 512.bit3 是故障跳闸标志，优先于 512.bit2 报警标志：bit3=1 时 515～523 按故障解析；否则在 bit2=1 时按报警解析。
- 非法 BCD、未知枚举和保留值只影响当前字段，不使整批失败。

## 读取区间

设备：256～258、268～270、336～343、352～357、432～433、512～523，加隐藏读取 787 额定电流序值。连接测试读取 256 一个寄存器。

故障：785 的低 8 位是类型（0=故障、1=报警、2=变位），高 8 位是记录编号 0～15，写入值为 `(记录编号 << 8) | 类型`。例如故障记录 3=`0x0300`、报警记录 3=`0x0301`、变位记录 3=`0x0302`。故障数据页当前不提供搜索条件，“立即读取”固定写入 `0x0000` 并读取故障记录 0。软件校验 06H 回显后读取 768～787；为兼容设备处理时间，可配置等待 0～2000 ms，默认 100 ms。表格展示/导出 768～787，其中 787 会按所选控制器映射成额定电流。

三种记录类型统一读取事件记录区：768～770 按 BCD 时间解析，771 按 5.4，772～779 按 5.5。选择类型 2 时也按相同地址读取和展示，不再视为缺少独立数据结构。软件以所选控制器解释 787，并用它计算相关电流值。根目录 README 的“待补充协议规则”是剩余问题清单。
