# 开发计划与实现状态

## 阶段 1：工程和技术基线（已完成）

- .NET 10 解决方案，中央包版本、Nullable、隐式 using、编译绑定。
- App/Core/Infrastructure/Simulator/Tests 五项目边界。
- Avalonia 12.1、ReactiveUI、Semi/Ursa、LiveCharts2 依赖预留、ClosedXML、Serilog。

## 阶段 2：协议核心（已完成）

- 抽象字节传输和 Modbus RTU 主站。
- 03H 批量读、06H 单寄存器写、CRC16、异常帧、严格地址/功能码/长度校验。
- 分段接收、超时/CRC 一次重试、设备异常不重试、全局异步锁。
- 寄存器目录、续寄存器合并、字序切换、倍率、BCD、状态/报警字段和协议警告。

## 阶段 3：基础设施和模拟器（已完成）

- 真实 `System.IO.Ports` 传输、动态端口目录。
- JSON 设置、四列 ClosedXML 导出、10,000 条内存日志与 14 天滚动文件。
- 模拟器 03H/06H、故障记录、动态测量值和错误注入。

## 阶段 4：桌面界面（已完成）

- 现代深蓝顶栏、连接侧栏、设备/录波/故障标签页、响应式滚动区域。
- 串口与设备分离状态、控件逐级启用、读取忙碌状态、主题和高级参数。
- 设备自动刷新；故障只手动读取；录波明确禁用。
- F12 日志窗口和系统保存文件对话框。

## 阶段 5：测试和交付（已完成基础验收）

- 协议、解析、导出与模拟器自动化测试。
- 解决方案零警告构建。
- 当前 macOS 完成实际窗口启动和浅色主题视觉检查。
- 自包含发布命令和现场实机清单已文档化；Windows/Linux 发布应在对应 OS 或 CI 复验。

## 实机到位后的阶段 6

1. 确认所有 uint32 的实际双字序；当前默认高字优先，可在高级设置切换，界面保留未实机确认警告。
2. 确认“×1 或 ×2 注1”、地址范围、785 就绪时间和记录序号新旧顺序。
3. 用逻辑分析/日志对照 TX/RX、设备显示值与 WireLink 结果。
4. 对超时、拔线、端口占用和设备异常执行现场恢复测试。
5. 获得录波协议后新增独立 `WaveformCatalog/Service/ViewModel`，启用 LiveCharts2，不侵入现有读取服务。
