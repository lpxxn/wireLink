
# 项目功能
通过串口读取设备信息，并进行解析，进行展示和导出Excel
设备是通过USB转-RS485接口连接的。

## 已确认协议规则

- `%` 字段直接使用寄存器原值，展示时在数值后追加 `%`，不再除以 100。
- 电流变比只可能为 1 或 2：
  - BW1：额定电流序值小于等于 10 时为 1，大于 10 时为 2。
  - BW3：额定电流序值小于等于 11 时为 1，大于 11 时为 2。
- 当前暂按“寄存器 787 返回额定电流序值”实现。界面由用户选择 `BW1 的控制器` 或 `BW3 的控制器`，软件根据控制器对应表把序值换算为额定电流，并计算电流变比。以 630A 为例，两种控制器的序值均为 4，变比均为 1。
- 5.5 中标注“×1 或 ×2 注1”的电流事件使用同一套“控制器系列 + 787 序值”规则。寄存器 788 不再参与电流变比计算。
- 上述 787 返回序值的规则属于当前实机前假设；连接设备后如发现 787 直接返回安培值，再按实机结果修改。
- 寄存器 785 的低 8 位指定记录类型（0=故障、1=报警、2=变位），高 8 位指定记录编号 0～15。编码公式为 `(记录编号 << 8) | 记录类型`。
  - 故障记录 3：`0x0300`。
  - 报警记录 3：`0x0301`。
  - 变位记录 3：`0x0302`。
- 故障数据页当前不显示记录类型和记录编号搜索条件，“立即读取”固定读取故障记录 0。06H 写入 785 并验证回显后读取 768～787；软件保留 0～2000 ms 可配置等待时间，默认 100 ms，作为通信兼容参数而不是 785 字段规则。
- 寄存器 512 的 bit3 是“故障跳闸标志”，bit2 是“报警标志”。bit3=1 时 515～523 按故障表解析；只有 bit3=0 且 bit2=1 时才按报警表解析。两个 bit 同时为 1 时仍优先按故障解析。
- 5.5.2 中的 `Int=516` 表示当前报警数据地址 516。报警事件只有数据 0 有效，517～523 为空；历史报警记录采用对应字段映射，只有 772 有效，773～779 为空。
- 5.5.1 的故障数据 3 暂不关联具体保护设置、倍率和单位；当前地址 519、历史地址 775 均直接显示寄存器十进制原始值。
- 当前没有故障或报警时，515 显示“无当前故障/报警”，516～523 留空且不显示协议警告。故障或变位事件中没有专门计算规则的数据字段直接显示十进制原始值。
- 故障记录时间和本次上电时间分别合并三个 BCD 寄存器，显示为 `yyyy-MM-dd HH:mm:ss`。
- 异常响应和设备专用通讯时序暂不继续补充。软件默认单次请求超时 1000 ms，可在高级通信设置中调整；超时后按通信失败处理。
- 设备地址由用户输入，界面允许 1～255；地址不正确时连接测试失败并提示通信错误，不再额外判断协议中的地址范围描述。
- 事件记录统一读取 768～779：768～770 按 BCD 时间解析，771 按 5.4，772～779 按 5.5。785 选择类型 2 时同样读取并展示这组地址，不再要求单独的“变位记录数据结构”。
- uint32 使用两个连续表格项时，排在前面的寄存器保存高 16 位，后面的寄存器保存低 16 位。例如 `[336,337]` 中 336 为高 16 位、337 为低 16 位；`[514,513]` 中 514 为高 16 位、513 为低 16 位。主程序固定使用高字优先，页面不再显示字序设置。

1. 最左边是连接设备配置，
* 要先选择串口，点击下拉框的时候，获取所有串口，并显示在下拉框中。再选择波特率，根据文档里的值写死就可以，点击`打开串口`按钮，会打开串口，更新`状态`为绿色，如果失败，则为红色。给出提示信息。
