
# 项目功能
通过串口读取设备信息，并进行解析，进行展示和导出Excel
设备是通过USB转-RS485接口连接的。
## 协议文档
协议文档 [tran.md](tran.md)
需要注意
* 表格的数据如果下一行只有`地址`值，没有其他值，  则需要使用上一行的`地址`值，说明这是一组，需要读取多个地址的数据。比如：
```
| 338          | 高精度电流测量 Ib    | uint32 | A    | R   | ×0.01     |
| 339          |                      |        |      |     |           |
```
* 表格的下一行没有`地址`值，则需要使用上一行的`地址`值，说明这一个包含两个数据。比如：

```
| 274          | 接地电流             | uint16 | A     | R   | ×电流变比 |
|              | 漏电电流             |        |       |     | ×0.01     |
```
```
| 1053 | H：功率需用值时间窗口值(char) | uint16 | min | R/W | 5～60,步长 1 |
|  | L：电流需用值时间窗口值(char) |  |  |  | 5～60,步长 1 |
```

## 待补充协议规则

以下问题尚不能从 [tran.md](tran.md) 和 [input.docx](input.docx) 中确定，需要补充协议规则后才能完成准确解析：

1. **uint32 双寄存器字序**
   - 已明确单个寄存器内部按高字节、低字节传输。
   - 未说明两个寄存器组合成 uint32 时哪个是高 16 位。
   - 需要特别确认 `[336, 337]`、`[432, 433]`、`[514, 513]` 等组合。
   - 软件实现约定：默认高字优先，并可在高级设置切换为低字优先；实机确认前保留警告。

2. **“×1 或 ×2 注1”的选择条件**
   - 5.5.1 和 5.5.2 均引用“注1”。
   - `input.docx` 中只有上标引用，没有对应脚注、尾注或正文说明。
   - 需要补充何种设备、框架等级或事件应乘 1，何种情况应乘 2。

3. **当前故障与当前报警同时存在时的解释规则**
   - 515～523 同时引用故障和报警解析表。
   - 协议未说明故障标志与报警标志同时有效时，这组寄存器表示哪一种事件。

4. **故障记录序号的新旧顺序**
   - 785 支持记录序号 0～15。
   - 未说明 0 是最新记录、最旧记录，还是设备内部槽位编号。
   - 界面暂时只显示“记录 0～15”，不标注新旧。

5. **写入 785 后的记录就绪时序**
   - 未说明写入记录类型和序号后是否可以立即读取 768～787。
   - 需要补充是否存在等待时间、状态位或轮询确认机制。

6. **变位记录的数据结构**
   - 785 定义了记录类型 2“变位记录”。
   - 协议没有说明变位记录在 768～779 中各字段的含义和解析方式。

7. **5.5.2 中 `Int=516` 的含义**
   - 需要确认这里表示当前报警数据起始地址 516，还是表格序号应为 0。
   - 同时需要确认该规则如何映射到历史记录地址 772～779。

8. **设备地址有效范围冲突**
   - 协议不同章节分别出现 0～255、1～99。
   - 需要确认实际可读取的从机地址范围，以及地址 0 是否仅用于广播。

9. **异常响应和通讯时序**
   - 文档只列出异常码 02、03、04，没有给出完整异常响应帧示例。
   - 未给出设备响应超时、建议重试间隔和单次 03H 最大寄存器数量。

10. **录波数据协议**
    - 尚缺少寄存器地址、触发与读取流程、三相排列、采样率、采样点数、数据类型、倍率、时间轴及分包规则。

> 已确认的 `% ÷ 100`、普通 `×N` 倍率及原始值/计算值日志规则不列入“待补充”。

## 页面


1. 最左边是连接设备配置，
* 要先选择串口，点击下拉框的时候，获取所有串口，并显示在下拉框中。再选择波特率，根据文档里的值写死就可以，点击`打开串口`按钮，会打开串口，更新`状态`为绿色，如果失败，则为红色。给出提示信息。
* 打开串口后，下面的设备信息才可以编辑和点击，选择地址，点击`连接测试`按钮，随便读取一个设备信息，如果成功则表示连接成功，状态变为绿色，如果失败，则为红色。给出提示信息。

LiveCharts。日志库你来选一个。如果有更好的库，你可以推荐。

* 页面要写的漂亮，要符合现代审美
* 要有详细的中文注释
* 要有详细的 md 文档，方便后续维护和更新
